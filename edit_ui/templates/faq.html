Frequently Asked Questions


What is Maptcha?

Maptcha is a web-based interface doing rough placements of historical maps on modern geography.

The first version of maptcha.org was launched by Mike Migurski in July 2011, using the 1905 Sanborn Fire Insurance Atlas as its first map series test case. The atlas contained about 700 pages of street level maps of San Francisco, and when it was launched, volunteers from around the world used the site to place each page on to a modern map of the city, all within a few days, which was a wonderful surprise!

The application was further developed in 2012 by Stamen Design in San Francisco, under a grant from TheÂ Bill Lane Center for the American West at Stanford University to support for the 2013Year of the Bay project, crowdsourcing a new history of the San Francisco Bay Area.

How do I get started?

There are 4 basic steps to uploading a set of maps to Maptcha:

  1. Select a set of historical maps
    Ensure the digital versions are somewhere public on the web. Public here just means somewhere online that isn't password-protected. Maptcha calls this set of maps an "atlas."
     
  2. Create a spreadsheet that describes each of the maps
    You'll need a row in the spreadsheet to describe each map to the specifications below. Save or export your spreadsheet as a CSV file and upload it somewhere public on the web.
     
  3. Tell Maptcha where that CSV file is
    Using the upload form, give your atlas a name, state your institutional affiliation, add a link to your CSV file, then hit Next.
     
  4. Provide some simple geographical hints
    Note any particular map features (street-level, coastlines etc) and city-name clues to help determine a bounding box that contains all the maps in the atlas. This will come in handy later for volunteers who are placing the maps because Maptcha will be able to show them the right general area.

What's an atlas?

Your old maps and the CSV file of basic metadata about them is what we call an atlas.

One of the reasons we think the first version of Maptcha worked so well is that volunteers could see the potential end of their work placing each page of the Sanborn atlas. Giving people a sense of progress of the task at hand is useful, compared with having to slog through an overwhelmingly large list of millions of old maps.

What's the best way to create my CSV?

It's easiest to work with a spreadsheet first. You can download an example CSV file to get started with, and have a look at the more detailed field-level specification on the upload page.

What fields do I need in my CSV for each map?

There are only 3 required fields for each map:

  1. Title of the map,
  2. Date the map was created, and
  3. Image_URL is the fully qualified URL that will direct Maptcha to the digital map image file.

There are lots of other suggested fields too. You can see the full list on the upload page.

Can I play with the map placement tool?

Definitely!

Do you have a naming convention I should use?

No. The only things Maptcha requires you use are: public images & CSV files and the 3 required fields for each map. Just use names that you'll recognize later.

Is there a file size limit on map images?

Not that we've hit yet. In fact, the bigger the original image, the better. It really helps for volunteers to be able to zoom in at high detail for some placement tasks.

What image file formats do you accept?

Maptcha can process .JPG, .TIF, .PNG and .GIF. If your images are any other format, the process will fail.

Where do I put my images and CSV so Maptcha can get to them?

You can put your files anywhere that's publicly accessible on the web. That means not on an intranet, not behind a password, not on your local hard drive. If Maptcha can't access any of the files, the process will fail.

You might like to look at using Dropbox. (We're not connected to them, we just think it's a good, free service that makes it super easy to share files.)
